# ReasonWeave Nomenclature

ReasonWeave is a relevant name: reasoning is the deliberate judgment layer, and weaving is the act of combining multiple work strands into one coherent result.

## Terms

- ReasonWeave Agent Suite: full future system.
- ReasonWeave Orchestrator: this Codex skill.
- Weave: coordinated execution plan.
- Strand: independent task or agent lane.
- Pass: parallel execution wave.
- Pattern: dependency map.
- Knot: blocker, dependency, ambiguity, or conflict.
- Packet: compressed handoff artifact.
- Proof Gate: verification checkpoint.
- Ledger: durable evidence or history artifact.
- Signal: essential extracted fact.
- Dense Mode: high-compression output mode.
- Clear Mode: normal safe communication mode.

## Reserved Future Names

- `reasonweave-agent-suite`: public skill repo.
- `reasonweave-codex-plugin`: Codex plugin.
- `reasonweave-context-mcp`: context and memory MCP.
- `reasonweave-compress-mcp`: deterministic compression MCP.
- `reasonweave-runner-mcp`: worker orchestration MCP.
- `reasonweave-proof-mcp`: verification and evidence MCP.

## Naming Rules

- Use ReasonWeave terms in public docs, file names, prompts, and examples.
- Do not use crowded or borrowed product names for new components.
- Do not name new modes after archived skills.
- Literal archived folder names may appear only in migration commands or recovery notes.
